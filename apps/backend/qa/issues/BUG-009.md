# BUG-009: Teste de validate-check-in não autenticava como admin (401 indevido)

**Severity:** Medium
**Priority:** P2
**Type:** Functional
**Status:** Resolved

## Environment

- **Build:** main @ 2dd985d (anterior à bateria de correção 2026-05-03)
- **OS:** N/A
- **Browser:** N/A
- **URL:** `PATCH /check-ins/:checkInId/validate` (test only)

## Summary

`validate-check-in.controller.business-flow-test.ts` chamava o endpoint sem header `Authorization`, recebendo `401` antes mesmo de exercitar o handler. Como a rota é `isProtected: true` + `onlyAdmin: true`, o teste validava o middleware de auth em vez da regra de negócio. O bug ofuscou BUG-006 (resposta com campo errado), que só apareceu após o fix da autenticação.

## Reproduction

```bash
pnpm --filter backend test:business-flow -- -t "validate-check-in"
# Antes: status esperado 200, recebido 401.
```

Observed before the fix:

- Falha 401 mascarava o bug de contrato real.

## Expected

Teste autentica como admin, executa o endpoint e valida o response da regra de negócio.

## Root cause

Setup do teste não criava o usuário admin nem gerava JWT antes da chamada HTTP, omissão na criação inicial.

## Fix

`apps/backend/src/check-in/infra/controller/validate-check-in.controller.business-flow-test.ts` — adicionado:

1. `createAndSaveUser({ role: "ADMIN" })` no `beforeEach`.
2. Geração de JWT via `AuthenticateUseCase`.
3. Header `Authorization: Bearer <token>` em todas as chamadas.

## Verification

- `pnpm --filter backend test:business-flow -- -t "validate-check-in"` → passa.

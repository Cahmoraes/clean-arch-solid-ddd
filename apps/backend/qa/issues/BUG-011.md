# BUG-011: Teste de fetch-users sem header Authorization em rota protegida

**Severity:** Medium
**Priority:** P2
**Type:** Functional
**Status:** Resolved

## Environment

- **Build:** main @ 2dd985d (anterior à bateria de correção 2026-05-03)
- **OS:** N/A
- **Browser:** N/A
- **URL:** `GET /users` (test only)

## Summary

`fetch-users.business-flow-test.ts` chamava o endpoint admin-only sem JWT, recebendo `401` antes do handler. Mascarava BUG-005 (response vazio), que só ficou visível depois deste fix.

## Reproduction

```bash
pnpm --filter backend test:business-flow -- -t "fetch-users"
# Antes: 401 Unauthorized.
```

Observed before the fix:

- Falha 401 mascarava bug de serialização.

## Expected

Teste autentica como admin, dispara `GET /users` e valida `users` + `pagination`.

## Root cause

Setup do teste não autenticava o usuário; o controller exige `isProtected + onlyAdmin`.

## Fix

`apps/backend/src/user/infra/controller/fetch-users.business-flow-test.ts`:

1. `createAndSaveUser({ role: "ADMIN" })`.
2. `AuthenticateUseCase.execute(...)` para gerar JWT.
3. Header `Authorization: Bearer <token>` em cada chamada.

## Verification

- `pnpm --filter backend test:business-flow -- -t "fetch-users"` → passa.
- Após fix do BUG-005, body completo passa nas assertions.

# BUG-010: Teste de user-profile sem campo `role` na assertion

**Severity:** Low
**Priority:** P3
**Type:** Functional
**Status:** Resolved

## Environment

- **Build:** main @ 2dd985d (anterior à bateria de correção 2026-05-03)
- **OS:** N/A
- **Browser:** N/A
- **URL:** `GET /me` (test only)

## Summary

`user-profile.business-flow-test.ts` usava `toEqual({...})` sem incluir o campo `role`, que é parte do contrato do endpoint. Após o fix do `additionalProperties: true` (BUG-005) e da serialização correta, o `role` passou a aparecer no body e o `toEqual` falhava por shape incompleto.

## Reproduction

```bash
pnpm --filter backend test:business-flow -- -t "user-profile"
# Antes: AssertionError — extra property "role" not expected.
```

Observed before the fix:

- Falha de assertion mascarando que o endpoint estava agora correto.

## Expected

Assertion reflete o contrato completo, incluindo `role: "MEMBER"`.

## Root cause

Cobertura de assertion incompleta na escrita original do teste.

## Fix

`apps/backend/src/user/infra/controller/user-profile.business-flow-test.ts` — incluído `role: "MEMBER"` no objeto esperado.

## Verification

- `pnpm --filter backend test:business-flow -- -t "user-profile"` → passa.

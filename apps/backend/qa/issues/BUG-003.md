# BUG-003: Webhook Stripe rejeitava requisições válidas com 400

**Severity:** Critical
**Priority:** P0
**Type:** Functional
**Status:** Resolved

## Environment

- **Build:** main @ 2dd985d (anterior à bateria de correção 2026-05-03)
- **OS:** N/A (backend Node.js)
- **Browser:** N/A
- **URL:** `POST /webhook/stripe`

## Summary

O endpoint do webhook do Stripe respondia `400 Bad Request` mesmo para requisições com assinatura válida. Como consequência, nenhum evento de billing era publicado na fila e toda a integração de assinatura (Subscription Created/Updated/Deleted) ficava silenciosamente quebrada em produção.

## Reproduction

```bash
# Antes do fix:
curl -X POST http://localhost:3333/webhook/stripe \
  -H "Stripe-Signature: <assinatura-válida>" \
  -H "Content-Type: application/json" \
  -d '{"id":"evt_test","type":"customer.subscription.created", ...}'
# → HTTP/1.1 400 Bad Request
# Body com erro AJV: "body should be string"
```

Observed before the fix:

- Toda chamada legítima do Stripe respondia 400.
- Mensagem nunca era publicada na fila RabbitMQ.
- `webhook-stripe.business-flow-test.ts` falhava de forma intermitente.

## Expected

- Requisição com assinatura válida → `200 OK` e evento publicado na fila.
- Requisição com assinatura inválida → `400` com mensagem clara.

## Root cause

O schema OpenAPI declarava `body: z.string()` para preservar o payload bruto necessário à verificação de assinatura. O Fastify, porém, parseia o JSON automaticamente em objeto antes do AJV; o validador então rejeitava porque o tipo recebido era `object`, não `string`. O handler do controller jamais era atingido.

## Fix

`apps/backend/src/subscription/infra/controller/stripe-webhook.controller.ts` — remoção da declaração `body: z.string()` no schema; validação de assinatura passou a usar o `rawBody` exposto pelo `fastify-raw-body` plugin já configurado no adapter, sem depender do schema.

## Verification

- `pnpm --filter backend test:business-flow -- -t "stripe"` → passa.
- `pnpm --filter backend test:business-flow` → 44/44.
- Curl manual com assinatura válida e inválida retornam 200 e 400 respectivamente.

## Impact

- **Users Affected:** 100% — qualquer cliente que tentasse assinar via Stripe.
- **Frequency:** Sempre.
- **Workaround:** Nenhum em produção; manualmente reprocessar via dashboard do Stripe.

Clean Architecture + DDD — Visão Geral do Sistema

Node.js · Fastify · Inversify IoC · Prisma · RabbitMQ · Redis · Stripe

Bounded Contexts (DDD)

  • • user/ — cadastro, perfil, senha, roles, ativação
  • • session/ — autenticação JWT, logout, refresh token
  • • gym/ — academias, busca por nome e proximidade
  • • check-in/ — check-ins com validação de distância e tempo
  • • subscription/ — assinaturas Stripe + webhooks

Infraestrutura & Dados

  • • PostgreSQL :5432 — banco de dados principal (Prisma ORM)
  • • Redis :6379 — cache de sessões e blacklist de tokens
  • • RabbitMQ :5672 — broker de mensagens assíncronas
  • • BullMQ Workers — processamento de filas em background
  • • Nginx :80 — reverse proxy + roteamento Docker

Padrões Arquiteturais

  • • Clean Architecture — Domain → Application → Infra
  • • Either monad — tratamento de erros sem exceções
  • • Domain Events — Observable + DomainEventPublisher
  • • Repository pattern — InMemory para testes, Prisma em prod
  • • Inversify IoC — injeção de dependência por decoradores
